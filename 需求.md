## 目的

通过LED灯显示Claude Code工作状态

## 产出

### 指示器控制项目

* 基于C++
* 基于idf 5.5.3
* 启动时，依次点亮LED1/LED2/LED3，每次等待200ms，之后关闭所有LED
  * LED1 红色
  * LED2 橙色
  * LED3 绿色
* 启动后广播低功耗蓝牙
  * 使用 `ClaudeCodeIndicator` + `蓝牙mac`作为广播名称
* 与电脑python脚本连接成功后保持连接状态
* 蓝牙没有请求时进入Light‑sleep，保持蓝牙连接，间隔4秒唤醒一次
* 收到蓝牙请求
  * 保活
    * 响应，防止断开
  * 工作中
    * 点亮LED2，橙色
    * 关闭LED3
  * 等待用户确认
    * 点亮LED2，橙色，闪烁
    * 关闭LED3
  * 工作完成
    * 点亮LED3，绿色
    * 关闭LED2
  * 配对确认
    * 点亮所有LED，绿色
  * 配对成功
    * 关闭所有LED
* 检测到低电量，电池电压 <= 3.5V
  * 点亮LED1，红色
* 检测到电池电压 > 3.5V
  * 关闭LED1

### 电脑端主程序

* 基于Python脚本
* 第一次运行，通过低功耗蓝牙连接esp32c3，连接成功后，发送`配对确认`请求
  * 弹出对话，让用户确认，他的指示器是否3个灯都亮绿色
  * 用户确认 是，则将该蓝牙广播名字记录到磁盘，并且发送`配对成功`请求
  * 之后每次启动都读取之前记录的蓝牙广播名字，只连接这个名字的蓝牙，其他蓝牙忽略
* 通过低功耗蓝牙发送消息给esp32c3
* 保持连接
  * 发送保活请求，间隔`10`秒
* 本地开启TCP端口`54321`
* 收到Claude Code通知程序消息后，通过蓝牙发送给esp32c3
* 提供.bat脚本，方便用户启动`电脑端主程序`
  * 只允许一个进程执行，多次运行则直接退出之后的程序

### Claude Code通知程序

* 基于Python脚本
* 使用Hook集成到Claude Code
* 通过Hook触发的消息类型
  * 工作中
  * 等待用户确认
  * 工作完成
* Hook触发后，将消息发给电脑端主程序
  * 使用TCP连接电脑端主程序`54321`端口，并发送消息

### 安装程序

* 基于Python脚本
* 将hook脚本添加到Claude Code配置中，方便用户使用，不需要用户手工配置
* 提供.bat运行`安装程序`

## 硬件

* led引脚：GPIO3
  * 3个LED串联
* led型号：ws2812b
* 电池ADC引脚：GPIO0
  * 电池分压50%
* 芯片型号：ESP32-C3-MINI-1-N4

## 要求

* 所有参数信息不要硬编码，通过变量或者宏定义
* 如果缺少python库，直接使用pip install安装
* esp32c3 LED控制使用`led_strip`
